## 👋 What is this?

**Resources-for-X** is my personal, ever-growing collection of resources, references, tools, and ideas gathered while exploring technologies and building real applications.

Think of it as my **second brain** — not a tutorial repo, not a course dump. Every link, note, and syllabus here was added because I actually needed it while working on something. If it's here, it earned its place.

> Building [TimeLocus](https://github.com/) — a productivity analytics platform with Pomodoro timers, focus heatmaps, and AI-driven insights — taught me that great engineers don't just write code. They build a system around how they learn.

This repo is that system.

## 🧭 Philosophy

Most resource repos are just bookmarks. This one is different:

- **Context-first** — every resource has a reason for being here
- **Pruned regularly** — dead links and outdated tools get removed
- **Built in public** — if it helped me, it'll probably help you too
- **Opinionated** — I tag things by how useful they *actually* are, not how popular they are

---

## 🚀 How to use this repo

1. **Browse by category** — use the folder structure or README links
2. **Search** — GitHub's search works great across markdown files (`t` to open file search)
3. **Star it** ⭐ — so you can find it when you need it most (usually at 2am debugging something)
4. **Fork it** — make it yours, adapt the structure to your own stack and goals

## 🤝 Contributing

This is a personal repo, but good resources are universal.

If you know something that belongs here — a tool, article, pattern, or technique — open an issue or PR with a brief note on *why* it's useful. Context matters more than the link itself.
